Results plus .cxi file (2017+)

Processing results are saved into a separate HDF5 file in parallel to the .cxi file.

Events are synchronised between both files (same stack position in both files corresponds to the same event)



Results files are much smaller than CXI files because they do not contain the entire image for each event:

-rw-rw----+ 1 barty xu       13G May 15 02:39 mfxn1116-r0040-c00.cxi

-rw-rw----+ 1 barty xu      269M May 15 02:39 mfxn1116-r0040-c00.h5



This has the following benefits:

  1. • Results and metadata can be worked on without needing to copy the (large) .cxi file

  2. • Image file complies more closely to standard CXI and Nexus files

  3. • Results file can be more flexibly defined to suit FEL data without having to force data into the Nexus standard

  4. • Benefits of a small number of large files are retained



HDF5 files can be read directly by any decent scientific data analysis software.  You need to know what’s where since HDF5 is a container for multiple data sets.


Useful:

  1. > h5ls -h

  2. usage: h5ls [OPTIONS] file[/OBJECT] [file[/[OBJECT]...]

  3. ....

 

REsults file (.h5)

Results are saved in an HDF5 containing the results of Cheetah processing


 

Image data is saved in the format of the Coherent X-ray Imaging Data Bank (CXIDB) which is loosely based on NeXus.

Splitting out the results and metadata means that CXI files are much more simply structured than before.  A small amount of results are saved so as to not immediately break compatibility with CrystFEL
